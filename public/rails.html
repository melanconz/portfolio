Bloccit (Reddit Clone) Zachary Melancon

Github

This application was as a project (along with other Rails based applications) while I studied at Bloc. It uses the many featues provided by Ruby on Rails to allow for and MVC application that has user authentication, data varification, data seeding for development, user profiles, user roles, it's own JSON API A user can create an account, create posts, comment on posts, upvote, downvote and favorite comments and posts, search posts based on tags.


Click on the images to enlarge them.

Home

If a user is not already logged in when reaching the site, the home page will ask the user to sing in or sign up for an account.

Sign Up/Sign In

Users can choose to sign up or sign in, and all information entered is validate, checked against user info already in database, and checked for security.

Topics

Even if auser is not signed in, they can browse topics and posts. However, they can not create a new topic, comment on posts, vote on posts etc.

Topic Page

When a topic is clicked, if the topic was created by them or if they are in the admin role, they can choose to edit or delete the topic. All signed in users can create posts associated with a topic and upvote or downvote posts associated with a topic.

Post Page

After a post is created, it can be commented on, voted on, favorited, and if the current user role is admin or the user created the post, it can be edited or deleted.

User Profile

All users have their own profile page that contains a list of posts they've made and their current score. From there, a user can delete any posts or topics they have made or add comments to those posts.
